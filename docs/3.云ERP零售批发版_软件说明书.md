## 1. 软件概述

云ERP（零售批发版）是一款面向零售批发企业、连锁门店、经销商的一体化管理软件，集成销售管理、采购管理、库存管理、财务管理、客户管理、供应商管理等核心模块，帮助企业实现业务流程数字化、运营管理精细化。

**主要设计目标：**
- 实现进销存全链路闭环管理
- 支持多仓库、多门店统一管理
- 提供多维度经营数据统计分析
- 支持多终端访问（PC端为主，移动端适配）

## 2. 系统架构

本系统采用前后端分离架构，包含以下层次：

### 2.1 前端架构
- **技术选型：** React 18 + TypeScript + Vite
- **UI框架：** Ant Design 5
- **状态管理：** Zustand（全局状态）+ React Query（服务端状态）
- **HTTP客户端：** Axios
- **图表组件：** ECharts + Ant Design Charts
- **路由：** React Router DOM v6
- **前端工程化：** ESLint + TypeScript

### 2.2 后端架构
- **框架：** NestJS（Node.js）
- **ORM：** Prisma
- **数据库：** PostgreSQL 16
- **认证：** JWT + Passport
- **数据校验：** class-validator + class-transformer
- **密码加密：** bcrypt

### 2.3 系统模块划分
- **认证模块（Auth）：** 登录、退出、JWT令牌管理
- **账户模块（Account）：** 用户账号、角色、权限管理
- **商品模块（Product）：** 商品档案、分类管理
- **客户模块（Customer）：** 客户档案、联系人管理
- **供应商模块（Supplier）：** 供应商档案管理
- **仓库模块（Warehouse）：** 仓库档案管理
- **采购模块（Purchase）：** 采购订单、入库、退货
- **销售模块（Sale）：** 销售订单、出库、退货
- **库存模块（Inventory）：** 库存查询、调拨、盘点、预警
- **财务模块（Finance）：** 收付款、应收账款、应付账款

## 3. 技术特点

### 3.1 TypeScript统一语言栈
前后端均采用TypeScript开发，提供强类型检查，显著提升代码可读性、可维护性和开发效率，降低运行时错误。

### 3.2 React组件化开发
前端采用React 18，组件化开发模式实现高效复用，配合Vite快速构建，提升开发体验和运行性能。

### 3.3 NestJS模块化后端架构
后端采用NestJS框架，模块化设计思路，每个业务模块（Module）包含Controller、Service、DTO、Entity，架构清晰，职责分明，便于维护和扩展。

### 3.4 Prisma ORM多租户支持
使用Prisma作为ORM工具，支持多租户数据隔离，提供类型安全的数据库操作，支持数据库迁移和种子数据管理。

### 3.5 JWT无状态认证
采用JWT（JSON Web Token）实现无状态身份认证，支持Token自动刷新，保证系统安全性。

### 3.6 多端适配
前端采用响应式布局设计，支持主流浏览器（Chrome、Edge、Firefox、Safari），适配不同屏幕尺寸。

## 4. 功能模块说明

### 4.1 销售管理

**销售订单：** 支持创建、编辑、审核、作废销售订单；支持选择客户、商品、数量、单价、折扣；自动计算金额；支持打印销售单据。

**销售出库：** 根据已审核销售订单生成出库单；支持扫描/选择仓库；记录出库商品明细；更新库存数据；支持部分出库。

**销售退货：** 支持对已出库销售单发起退货；退货单审核后自动回滚库存；支持部分退货；记录退货原因。

### 4.2 采购管理

**采购订单：** 支持创建、编辑、审核、作废采购订单；支持选择供应商、商品、数量、单价；自动计算采购金额。

**采购入库：** 根据已审核采购订单生成入库单；支持多仓库入库；记录入库商品明细；自动更新库存。

**采购退货：** 支持对已入库采购单发起退货；退货审核后自动扣减库存；记录退货原因。

### 4.3 库存管理

**库存查询：** 支持按仓库、商品、分类多维度查询库存；实时显示库存数量和金额；支持库存预警设置。

**库存调拨：** 支持仓库间商品调拨；生成调拨单；调出仓和调入仓分别记录；自动更新库存账。

**库存盘点：** 支持定期盘点任务创建；支持按仓库/全量盘点；盘盈盘亏自动处理；生成盘点报表。

**库存预警：** 支持设置商品最低/最高库存预警线；低于或高于预警值时自动提醒；支持按类别批量设置。

### 4.4 财务管理

**收款管理：** 记录客户付款信息；支持按销售单核销；支持预收款管理；生成客户对账单。

**付款管理：** 记录对供应商付款信息；支持按采购单核销；支持预付款管理。

**应收账款：** 自动汇总未收款销售单；支持账龄分析；支持催款提醒。

**应付账款：** 自动汇总未付款采购单；支持账龄分析。

### 4.5 基础资料管理

**商品管理：** 商品档案维护；支持多级分类；记录商品编码、名称、规格、单位、零售价、采购价；支持商品图片管理。

**客户管理：** 客户档案维护；记录客户名称、联系人、电话、地址、信用额度；支持客户分类管理。

**供应商管理：** 供应商档案维护；记录供应商名称、联系人、电话、地址；支持供应商分类。

**仓库管理：** 仓库档案维护；支持设置默认仓库；支持仓库负责人管理。

**人员/账户管理：** 系统用户管理；支持多角色（管理员、业务员、财务员）；支持启用/禁用账户。

## 5. 系统界面

### 5.1 登录页面
系统提供统一的登录入口，输入用户名和密码后进入主界面。登录页展示系统名称"云ERP"，支持记住密码功能。

### 5.2 工作台/首页
登录成功后进入工作台，展示关键经营数据概览，包括：今日销售额、本月采购额、库存预警数量、应收账款金额等关键指标图表。

### 5.3 销售管理界面
支持销售订单列表展示，可按客户、日期、状态筛选；支持新增销售订单表单；支持销售单详情查看和打印。

### 5.4 采购管理界面
支持采购订单列表展示，可按供应商、日期、状态筛选；支持新增采购订单；支持采购单审核流程。

### 5.5 库存管理界面
库存列表展示各仓库商品实时库存；支持按仓库、商品分类筛选；支持库存调拨和盘点操作入口。

### 5.6 财务管理界面
收款/付款列表展示收付款记录；支持按客户/供应商核销；提供应收账款和应付账款汇总报表。

## 6. 安装与部署

### 6.1 前端部署
```bash
# 安装依赖
pnpm install

# 开发环境启动
pnpm dev

# 生产环境构建
pnpm build
```

前端构建产物为静态资源，可部署至Nginx或其他Web服务器。

### 6.2 后端部署
```bash
# 安装依赖
pnpm install

# 生成Prisma Client
pnpm prisma:generate

# 数据库迁移
pnpm prisma:migrate

# 启动服务（开发）
pnpm dev

# 启动服务（生产）
pnpm build && pnpm start
```

### 6.3 环境要求
- **Node.js：** 18+
- **数据库：** PostgreSQL 16+
- **包管理器：** pnpm（推荐）或 npm
- **前端：** Chrome/Edge/Firefox/Safari 最新版本

## 7. 使用说明

### 7.1 初始化步骤
1. 部署系统后，使用管理员账号登录（默认账号：admin，密码：admin123）
2. 首次登录后建议修改默认密码
3. 在"基础资料"中维护商品档案、客户档案、供应商档案、仓库档案
4. 根据业务需要创建系统用户并分配角色权限

### 7.2 日常业务流程
**销售流程：** 客户建档 → 录入销售订单 → 审核 → 出库 → 收款核销

**采购流程：** 供应商建档 → 录入采购订单 → 审核 → 入库 → 付款核销

**库存管理：** 定期盘点 → 库存调拨（如需）→ 关注预警提醒

### 7.3 数据报表
系统提供多维度经营数据统计，支持按日/周/月/年查看销售报表、采购报表、库存报表、财务报表，辅助经营决策。
